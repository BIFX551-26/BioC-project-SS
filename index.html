Bioconductor simpleSingleCell protocol for scRNA-seq

What is simpleSingleCell?

  • A family of Bioconductor workflows for low-level scRNA‑seq analysis by Lun, McCarthy & Marioni (2016).
  • Implemented entirely in R/Bioconductor around the SingleCellExperiment container.
  • Now archived; content lives at OSCA
  • Covers: QC → normalization → HVG selection → dimensionality reduction → clustering → marker genes.

Objective

From reads to a count matrix

  • Single-cell RNA sequencing (scRNA-seq) is widely used to profile the transcriptome of individual cells.
  • Single-cell RNA-seq starts as millions of sequencing reads that must be summarized into a gene × cell expression matrix.
  • Typically, each row is a gene and each column is a cell; entries are the number of reads or UMIs assigned to that gene in that cell.
  • UMI counts are treated like read counts but are less affected by PCR amplification, giving a cleaner measure of the number of captured molecules.
  • Once this quantification is done, all downstream QC, normalization, and analysis in R/Bioconductor operate on this count matrix.

Fig 1: Schematic of a typical scRNA-seq analysis workflow. Each stage (separated by dashed lines) consists of a number of specific steps, many of which operate on and modify a SingleCellExperiment instance.

3Protocols for 3Dataset - ‘simpleSingleCell’ workflow pipeline

3 Protocol’s : sc-RNA Analysis following count matrix.

Constructing the count matrix depends strongly on the experimental protocol:

1. Droplet-based protocols

  • Examples: 10X Genomics, inDrop, Drop-seq.

  • Many cells, 3′ UMI counts, shallow coverage per cell.

  1. Plate-based protocols with UMIs

  • Examples: CEL‑seq2, MARS‑seq.

  • Fewer cells, UMI counts, deeper coverage per cell.

3. Plate-based protocols with full-length reads

  • Examples: Smart‑seq2, sciRNA‑seq.

  • High coverage per cell, non-UMI read counts.

3-Dataset using ‘simpleSingleCell’ workflow pipeline:

Goal: Comparing three scRNA‑seq protocols and biological systems using a common Bioconductor workflow ’simpleSingleCell`.

In this project we analyze one dataset from each category

Datasets & questions:

  1. Paul et al. Mouse HSCs Measured with (MARS‑seq) - (Plate + UMI).
  • Mouse haematopoietic stem & progenitor hierarchy.
  • Question: how does transcription change along the myeloid–erythroid differentiation trajectory?
  1. Bach et al. Mouse mammary Epithelial cells with (10X Genomics) - (Droplet + UMI)

  • Mouse mammary epithelium and microenvironment.

  • Question: what cell types and states are present in the mammary gland during gestation?

  1. Messmer et al. Human hESC with (Smart‑seq2) - (Plate + Reads full-length)

  • Human embryonic stem cells, naive and primed states plus early differentiation.

  • Question: how heterogeneous are naive/primed hESCs and early lineages at single‑cell resolution?

All three are analyzed with an OSCA/simpleSingleCell-style pipeline.

Workflow overview

Raw counts (protocol-specific)
        ↓
   Quality Control       ← remove low-quality cells & outliers
        ↓
   Normalization         ← scran deconvolution size factors + logNormCounts
        ↓
Variance Modelling       ← modelGeneVar(…)/modelGeneVarByPoisson/WithSpikes
        ↓
HVG Selection            ← choose genes with variance above technical trend
        ↓
Dimensional Reduction    ← PCA, t-SNE / UMAP
        ↓
Graph-based Clustering   ← SNN graph → Walktrap / Louvain
        ↓
Marker Gene Detection    ← findMarkers to define cluster identities

Core idea: same pipeline across protocols, with protocol-specific tweaks.

1. Paul mouse HSC (MARS‑seq)

1.1 Data & protocol

  • Plate-based MARS‑seq UMI protocol
  • It is a Low-throughput but high-depth protocol targeting Mouse Bone marrow Hematopoietic stem and progenitor cells under multiple treatments.
  • Moderate numbers of cells per plate; UMI counts, shallow coverage per cell but good for trajectories.

Note

Data loading:

library(scRNAseq)
sce.paul <- PaulHSCData(ensembl=TRUE)

Biological question :

Q: How do Transcriptional programs change along the HSC → Myeloid/Erythroid differentiation trajectory?

Methods (what & how)

1.2 QC(Quality Control):

  • Per-cell QC metrics: library size, detected genes; limited mitochondrial annotation.
  • Plate-aware QC: Identified “bad” plates with unusually low thresholds, then recomputed QC excluding them.

Note

QC:

library(scater)
stats <- perCellQCMetrics(sce.paul) 
qc <- quickPerCellQC(stats, batch=sce.paul$Plate_ID)

# Detecting batches with unusually low threshold values.
lib.thresholds <- attr(qc$low_lib_size, "thresholds")["lower",]
nfeat.thresholds <- attr(qc$low_n_features, "thresholds")["lower",]
ignore <- union(names(lib.thresholds)[lib.thresholds < 100],
          names(nfeat.thresholds)[nfeat.thresholds < 100])

1.3 Normalization & HVGs:

  • Deconvolution size factors (computeSumFactors) and logNormCounts.
  • Poisson-based mean–variance modeling (modelGeneVarByPoisson) → top ~10% HVGs.

Note

  • denoisePCA → t‑SNE on HVGs.
  • SNN graph (buildSNNGraph) + community detection (cluster_louvain / cluster_walktrap).

1.5 Markers:

  • findMarkers per cluster; interpret using known myeloid/erythroid markers.

1.6 Results (what we learned):

  • QC removed cells with extremely low library sizes and detected genes, especially on problematic plates.
  • Poisson-based HVG selection captured genes varying along lineage trajectories rather than pure noise.
  • t‑SNE revealed gradient-like manifolds instead of perfectly discrete clusters, reflecting continuous differentiation.
  • Clusters corresponded to early HSCs vs more committed progenitors, with markers such as Gata1, Mpo, Elane distinguishing branches.

Take‑home: MARS‑seq is well-suited for mapping continuous differentiation trajectories when combined with careful plate-aware QC and Poisson variance modeling.

2. Bach Mouse mammary (10X Genomics)

2.1 Data & protocol

  • The Bach mouse mammary dataset profile mammary epithelial cells (and associated stromal/immune cells) using the High-throughput 10X Genomics droplet-based 3′ UMI protocol scRNA‑seq.
  • The goal is to chart the hierarchy of mammary epithelial lineages (basal, luminal progenitors, mature luminal) and their responses across developmental stages
  • Many Mouse mammary epithelial cells, plus stromal and immune cells from gestation - shallow coverage per cell.

Note

Data Loading

library(scRNAseq) 
sce.mam <- BachMammaryData(samples="G_1")

Biological question

Q: What epithelial subtypes and stromal/immune populations exist in the mammary gland during gestation?

Methods (what & how)

2.2 QC

  • Per-cell metrics including library size, detected features, and mitochondrial percentage using perCellQCMetrics.
  • quickPerCellQC used thresholds on library size, detected genes, and mitochondrial percent to discard low-quality cells.

Note

QC

is.mito <- rowData(sce.mam)$SEQNAME == "MT" 
stats <- perCellQCMetrics(sce.mam, subsets=list(Mito=which(is.mito))) 
qc <- quickPerCellQC(stats, percent_subsets="subsets_Mito_percent") 
sce.mam <- sce.mam[,!qc$discard]

2.3 Normalization & HVGs:

  • Deconvolution size factors (computeSumFactors) and logNormCounts.
  • Poisson-based variance modeling (modelGeneVarByPoisson) → top ~10% HVGs.

2.5 Markers:

  • findMarkers used to identify cell-type markers and annotate clusters as basal, luminal progenitors, mature luminal, fibroblasts, immune cells, etc.

2.6 Bach mammary – Results (what we learned)

  • QC effectively removed low-count, high-mito cells, visible as a low-count/high-mito tail in QC plots.
  • Normalization and Poisson-based HVG detection highlighted genes with strong biological variance across mammary populations.
  • t‑SNE showed clearly separated clusters, unlike the gradient seen in Paul HSC:
    • Basal cells, luminal progenitors, mature luminal cells, stromal fibroblasts, and immune populations.
  • Cluster markers matched canonical genes such as Krt5 (basal), Krt8/18 (luminal), Esr1 (hormone-responsive), and immune markers.

Take‑home: 10X excels at mapping diverse discrete cell types in complex tissues, with standard QC and deconvolution normalization.

3. 1 Data & protocol

  • Plate-based Smart‑seq2 full-length protocol.
  • Human embryonic stem cells in naive and primed pluripotent states, plus early differentiation states.
  • Fewer cells but high read depth and sensitivity per cell.

Note

Biological question

Q: How heterogeneous are naive vs primed hESCs and early differentiation trajectories at the transcriptomic level?

3.B Messmer hESC – Methods (what & how)

3.2 QC:

  • Per-cell library size, detected genes, mitochondrial percentage, and ERCC spike‑in percentages across experiment batches and phenotypes.
  • quickPerCellQC and related filters used to remove low-quality cells per batch.

Note

QC

sce.mess <- addPerCellQCMetrics(sce.mess, 
                                subsets=list(Mito=grep("^MT-", 
                                                  rowData(sce.mess)$SYMBOL)))

# Define filtering thresholds and identify low-quality cells
qc.stats <- perCellQCMetrics(sce.mess)
reasons <- perCellQCFilters(sce.mess, 
                            sub.fields=c("subsets_Mito_percent", "altexps_ERCC_percent"))

3.3 Normalization & variance modeling:

  • Deconvolution size factors (computeSumFactors) and logNormCounts.
  • Spike‑in–aware variance modeling (modelGeneVarWithSpikes) with blocking by experiment batch to separate technical noise from biological variation.

Note

NORM.

library(scran)

set.seed(10000)
clusters <- quickCluster(sce.mess)
sce.mess <- computeSumFactors(sce.mess, cluster=clusters)
sce.mess <- logNormCounts(sce.mess)

3.4 Cell cycle & batch correction:

  • Cell cycle phase assignment using cyclone with human marker pairs.
  • Regression-based correction (correctExperiments with RegressParam) including phenotype and batch covariates.
  • Contrastive PCA and sparse cPCA (scPCA) using primed-only cells as background to reduce cell-cycle-driven variation.

Note

C.C/B.C

hs_pairs <- readRDS(system.file("exdata", "human_cycle_markers.rds", package="scran"))
assigned <- cyclone(sce.mess, pairs=hs_pairs, 
                    gene.names=rownames(sce.mess),
                    BPPARAM=BiocParallel::MulticoreParam(10))
sce.mess$phase <- assigned$phases

dec <- modelGeneVarWithSpikes(sce.mess, "ERCC", block = sce.mess$`experiment batch`)

3.6 Messmer hESC – Results (what we learned)

  • QC combining mitochondrial and ERCC metrics removed low-quality cells evenly across batches and phenotypes.
  • Spike‑in–aware variance modeling identified HVGs that separate naive and primed states instead of just capturing technical noise.
  • Cell cycle was a major driver of variation in naive and primed cells; cPCA/scPCA reduced phase-based separation while maintaining naive vs primed structure, confirmed by reduced silhouette widths for phases.
  • Clusters showed enrichment of pluripotency markers (POU5F1/OCT4, NANOG, SOX2) in pluripotent clusters and lineage markers in differentiating subsets.

Take‑home: Smart‑seq2 plus spike‑ins and cPCA allow fine-grained analysis of subtle pluripotent state differences and early lineage specification.

Aspect Paul HSC (MARS‑seq) Bach mammary (10X) Messmer hESC (Smart‑seq2)
Protocol Plate-based UMI Droplet-based 3′ UMI Plate-based full-length
Organism / system Mouse HSC & progenitors Mouse mammary + stroma Human ESCs & early lineages

Variance modeling Poisson trend Poisson trend Spike‑in + batch-aware trend

Challenges

  • Managing Bioconductor data packages on a very recent R/Bioconductor release (AnnotationHub, scRNAseq, HCAData) required extra care.

-Namespace conflicts ex: dplyr’s select() vs AnnotationDbi’s select() for EnsDb objects; solved by using AnnotationDbi::select().

  • Protocol-specific design quirks especially Plate issues in Paul HSC, high mitochondrial tails in Bach 10X, strong cell cycle signals in Messmer hESC.

-Interpretation of graphs was a challenge mainly Continuous gradients (Paul) vs discrete clusters (Bach, Messmer) require different visualization and annotation strategies.

Lessons learned

  • Installing Bioconductor dependencies with compatible versions
  • Matching gene symbols to Ensembl IDs for the cyclone classifier
  • Choosing appropriate QC thresholds — MAD-based thresholds adapt to the data but require biological judgment
  • t-SNE is stochastic — set a seed for reproducibility

Conclusion

Using a common Bioconductor single-cell workflow, it is possible to analyze and interpret diverse single-cell datasets—ranging from hematopoietic progenitors to mammary gland tissue to embryonic stem cells—by adapting a few key steps to the underlying protocol and biological context. For each dataset: • Paul HSC illustrates how scRNA‑seq can reconstruct continuous differentiation manifolds and identify progenitor hierarchy. • Bach mammary shows how high-throughput droplet data reveal multiple discrete cell populations and tissue organization. • Messmer ESC demonstrates how plate-based full-length protocols enable detailed characterization of pluripotent and early differentiation states. The step-by-step workflow (QC, normalization, HVGs, DR, clustering, marker analysis) is conceptually identical across all three, while protocol-specific choices (e.g., normalization strategy) and interpretation focus differ.
