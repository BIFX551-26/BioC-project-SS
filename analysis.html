Comparing scRNA-seq Protocols with Bioconductor simpleSingleCell wokflow

Paul et al. Mouse HSC (MARS-seq), Bach et al. Mouse mammary (10x Genomics), Messmer et al. Human ESC (Smart-seq2)

Author

Saifun Nahar, Arman Moshasha, Suleman Mohammad

Published

May 9, 2026

Overview

Sequencing data from single-cell RNA-seq experiments must be converted into a matrix of expression values. This is usually a count matrix containing the number of reads mapped to each gene (row) in each cell (column). Alternatively, the counts may be that of the number of unique molecular identifiers (UMIs); these are interpreted in the same manner as read counts but are less affected by PCR artifacts during library preparation (Islam et al. 2014).

Constructing a count matrix from raw scRNA-seq data encompasses a variety of different experimental protocols.

Generally followed Protocols in Present day scRNA-seq experiments can be broadly categorized as follows:

  1. Droplet-based protocols : 10X Genomics, inDrop and Drop-seq;
  2. .Plate-based protocols with UMIs : CEL-seq2 and MARS-seq
  3. Plate-based protocols with reads : Smart-seq2 and others like sciRNA-seq

Each approach requires a different processing pipeline to deal with cell Demultiplexing and UMI Deduplication (if applicable).

This document is an effort to combine these 3 protocols in one place for easy analysis while pointing out differences in each individual protocols . For this analysis we selected three different pipelines namely 10x Genomics, MARS-seq and Smart-seq2 representing one from each protocol.

This document will briefly describe different methods used to generate a count matrix and read it into R, different approaches followed for single-cell RNA‑seq analyses while each using a different protocol, dataset and biological system.

  1. Paul et al. Mouse Hematopoietic Stem and Progenitor cells (HSCs) measured with MARS‑seq.
  2. Bach et al. Mouse mammary Epithelial and Stromal cells profiled with 10x Genomics.
  3. Messmer et al. Human embryonic stem cells (hESCs) profiled with Smart‑seq2, including differentiation states.

All three analyses follow the same core steps:

1. Data loading and basic inspection

2. Per-cell quality control (QC)

3. Normalization

4. Variance Modelling and highly variable gene (HVG) selection

5.Dimensionality reduction (PCA, UMAP)

6. Graph-based clustering

7. Marker detection and biological interpretation

Each section is self‑contained but uses the same core Bioconductor tools so you can compare workflows across protocols.

1.3 Quality control

unfiltered <- sce.paul

For some reason, only one mitochondrial transcripts are available, so we will perform quality control using only the library size and number of detected features. Ideally, we would simply block on the plate of origin to account for differences in processing, but unfortunately, it seems that many plates have a large proportion (if not outright majority) of cells with poor values for both metrics. We identify such plates based on the presence of very low outlier thresholds, for some arbitrary definition of “low”; we then redefine thresholds using information from the other (presumably high-quality) plates.

library(scater)
stats <- perCellQCMetrics(sce.paul) 
qc <- quickPerCellQC(stats, batch=sce.paul$Plate_ID)

# Detecting batches with unusually low threshold values.
lib.thresholds <- attr(qc$low_lib_size, "thresholds")["lower",]
nfeat.thresholds <- attr(qc$low_n_features, "thresholds")["lower",]
ignore <- union(names(lib.thresholds)[lib.thresholds < 100],
    names(nfeat.thresholds)[nfeat.thresholds < 100])

# Repeating the QC using only the "high-quality" batches.
qc2 <- quickPerCellQC(stats, batch=sce.paul$Plate_ID,
    subset=!sce.paul$Plate_ID %in% ignore)
sce.paul <- sce.paul[,!qc2$discard]

3. “Smart‑seq2:” Messmer et al. Human Embryonic Stem-cells (hESC)

3.1 Introduction:

Smart-seq2 is the sequencing method used for the Messmer (et al. 2019)[3] human embryonic stem cell (hESC) Messmer dataset. The analysis is performed using the Bioconductor package simple SingleCellExperiment, which is designed for handling single-cell RNA sequencing data. The workflow includes several steps such as data preprocessing, quality control, normalization, dimensionality reduction, clustering, and differential expression analysis. The goal of the analysis is to identify distinct cell populations and understand their gene expression profiles in the context of human embryonic stem cells.

3.2 Data loading:

The dataset is loaded into R using the readRDS function, which reads the serialized R object containing the single-cell data. The data is stored in a SingleCellExperiment object, which allows for efficient handling and analysis of single-cell RNA-seq data. The dataset includes information about the experiment batch and phenotype of the cells, which will be used for downstream analyses.

library(scRNAseq)

sce.mess <- MessmerESCData()
sce.mess$`experiment batch` <- factor(sce.mess$`experiment batch`)
library(AnnotationHub)

ens.hs.v97 <- AnnotationHub()[["AH73881"]]
anno <- AnnotationDbi::select(ens.hs.v97, keys=rownames(sce.mess), 
    keytype="GENEID", 
    columns=c("SYMBOL"))

rowData(sce.mess) <- anno[match(rownames(sce.mess), anno$GENEID),]

3.3 Quality control

Quality control is performed to filter out low-quality cells and genes. This typically involves calculating metrics such as the number of detected genes, total counts per cell, and the percentage of mitochondrial gene expression. Cells that do not meet certain thresholds for these metrics are removed from the analysis to ensure that only high-quality data is used for downstream analysis.

3.4 Normalization

Normalization is performed to account for differences in sequencing depth and other technical variations across cells. This step typically involves scaling the counts to a common library size and applying a transformation to stabilize the variance. Normalization allows for more accurate comparisons of gene expression levels across cells and is essential for downstream analyses such as clustering and differential expression.

library(scran)

set.seed(10000)
clusters <- quickCluster(sce.mess)
sce.mess <- computeSumFactors(sce.mess, cluster=clusters)
sce.mess <- logNormCounts(sce.mess)

3.5 Cell cycle phase assignment

Cell cycle phase assignment is performed to classify cells into different phases of the cell cycle (G1, S, G2/M) based on the expression of known cell cycle marker genes. This can provide insights into the proliferative state of the cells and help to identify potential sources of variation in gene expression that are related to cell cycle effects.

set.seed(10001)
hs_pairs <- readRDS(system.file("exdata", "human_cycle_markers.rds", package="scran"))
assigned <- cyclone(sce.mess, pairs=hs_pairs, 
                    gene.names=rownames(sce.mess),
                    BPPARAM=BiocParallel::MulticoreParam(10))
sce.mess$phase <- assigned$phases

3.6 Feature selection

Feature selection is performed to identify highly variable genes that are informative for downstream analyses such as clustering and dimensionality reduction. This typically involves calculating the variance of gene expression across cells and selecting genes that exhibit higher variability than expected based on their mean expression levels. The selected genes are then used for subsequent analyses to capture the biological heterogeneity in the dataset.

dec <- modelGeneVarWithSpikes(sce.mess, "ERCC", block = sce.mess$`experiment batch`)
top.hvgs <- getTopHVGs(dec, prop = 0.1)
par(mfrow=c(1,3))
for (i in seq_along(dec$per.block)) 
{
current <- dec$per.block[[i]]
plot(current$mean, current$total, 
        xlab="Mean log-expression", 
        ylab="Variance", pch=16, cex=0.5, main=paste("Batch", i))
fit <- metadata(current)
points(fit$mean, fit$var, col="red", pch=16)
curve(fit$trend(x), col='dodgerblue', add=TRUE, lwd=2)
}

3.7 Batch correction

Batch correction is performed to remove technical variations that arise from differences in experimental conditions or processing batches. This step is crucial for integrating data from multiple batches and ensuring that the observed biological differences are not confounded by batch effects. Methods such as ComBat, MNN (Mutual Nearest Neighbors), or Harmony can be used for batch correction, which adjust the gene expression values to mitigate the influence of batch-specific biases.

library(batchelor)
sce.mess <- correctExperiments(sce.mess, 
            PARAM = RegressParam(
                     design = model.matrix(~sce.mess$phenotype + 
                                            sce.mess$`experiment batch`),
                    keep = 1:2 )
                              )

3.8 Dimensionality reduction

Dimensionality reduction techniques such as Principal Component Analysis (PCA) and t-distributed Stochastic Neighbor Embedding (t-SNE) are applied to the normalized data to visualize the high-dimensional gene expression data in a lower-dimensional space. This helps to identify patterns and clusters of cells based on their gene expression profiles. PCA reduces the dimensionality by finding the directions of maximum variance in the data, while t-SNE focuses on preserving local relationships between cells.

library(matrixStats)
library(scater)
library(MatrixGenerics)
library(BiocSingular)
library(gridExtra)
library(scuttle)
rowVars <- function(x, ..., useNames = TRUE) {
  matrixStats::rowVars(x, ...)
}
set.seed(1101001)
sce.mess <- runPCA(sce.mess, 
                   subset_row = top.hvgs, 
                   exprs_values = "corrected", 
                   BSPARAM = BiocSingular::IrlbaParam())

sce.mess <- runTSNE(sce.mess, dimred = "PCA", perplexity = 40)

From a naive PCA, the cell cycle appears to be a major source of biological variation within each phenotype. To mitigate this, we can regress out the cell cycle scores from the PCA.

gridExtra::grid.arrange(
    plotTSNE(sce.mess, colour_by = "phenotype") + ggtitle("By phenotype"),
    plotTSNE(sce.mess, colour_by = "experiment batch") + ggtitle("By batch "),
    plotTSNE(sce.mess, colour_by = "CDK1", swap_rownames="SYMBOL") + ggtitle("By CDK1"),
    plotTSNE(sce.mess, colour_by = "phase") + ggtitle("By phase"),
    ncol = 2
)

Note: In principle, Contrastive PCA cPCA and Sparse PCA scPCA could further reduce cell‑cycle effects using primed-only cells as background, as shown in the OSCA Messmer workflow, but here we only focused on standard PCA/t‑SNE on batch‑corrected HVGs.

Interpretation: clusters correspond to pluripotent ESC states and early differentiation trajectories, based on markers such as POU5F1 (OCT4), NANOG, SOX2 for pluripotency and lineage markers for differentiation.

4. Cross-dataset comparison (high level)

4.1 Protocol differences

  • MARS‑seq (Paul HSC)
    • Plate-based UMI protocol capturing mouse haematopoietic stem and progenitor cells.
    • Moderate cell numbers per plate, UMI counts with relatively shallow coverage per cell.
  • 10X Genomics (Bach mammary)
    • Droplet-based 3′ UMI protocol capturing many mammary epithelial and stromal cells during gestation.
    • Very large cell numbers with shallow per-cell coverage.
  • Smart‑seq2 (Messmer hESC)
    • Plate-based full-length read protocol applied to human embryonic stem cells in naive and primed states.
    • Fewer cells but high read depth and sensitivity, enabling detection of lower expressed genes and isoforms.

4.2 QC and normalization choices

  • Paul HSC (MARS‑seq)
    • QC used library size and detected genes, with extra plate-aware logic to identify problematic plates before recomputing thresholds.
    • Normalization based on scran’s deconvolution size factors and logNormCounts.
  • Bach mammary (10X)
    • QC combined library size, detected features, and mitochondrial percentage with quickPerCellQC, reflecting typical 10X QC practice.
    • Normalization again used deconvolution size factors, confirming that the approach scales to large UMI datasets.
  • Messmer hESC (Smart‑seq2)
    • QC used mitochondrial and ERCC spike-in proportions across experiment batches and phenotypes, exploiting the high-coverage, spike‑in–rich design.
    • Normalization leveraged scran’s size factors and compared endogenous and spike‑in size factors to ensure consistent scaling.

4.3 HVGs, dimensionality reduction and clustering

  • HVG selection
    • Paul and Bach used Poisson-based variance modeling (modelGeneVarByPoisson) to define HVGs from endogenous genes.
    • Messmer used spike‑in‑aware variance modeling (modelGeneVarWithSpikes) with batch blocking, taking advantage of ERCC controls.
  • Dimensionality reduction
    • Paul and Bach used denoisePCA and t‑SNE on HVGs to visualize major cell populations and trajectories.
    • Messmer used PCA and t‑SNE on batch‑corrected data, with additional contrastive PCA and sparse cPCA to reduce cell-cycle-driven variation.
  • Clustering and marker detection
    • All three datasets used SNN graphs (buildSNNGraph) and Walktrap/Louvain clustering from igraph, followed by findMarkers to identify genes distinguishing clusters.
    • In Paul HSC, clusters captured myeloid–erythroid differentiation; in Bach, mammary cell types; in Messmer, naive/primed hESC states and differentiation trajectories.
